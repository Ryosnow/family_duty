# 全项目审查问题修复方案

## 目标

修复全项目审查中确认的任务改期重复生成、单次覆盖丢失、通知跨日失效、报表日期跳转、首页进度统计、历史计数文案、备份校验、无障碍标签和 UI 测试误判问题。

## 设计

- 为固定任务实例增加可选的原始发生日期和显式单次覆盖标记。旧数据缺少原始日期时回退到当前计划日期；旧调整数据继续通过调整说明、Deadline 和分值差异识别。
- 任务生成使用“规则 ID + 原始发生日期”去重，展示和单次改期继续使用 `scheduledDate`。
- 通知改为按未来具体日期创建不重复请求，滚动安排最多未来 28 天，避免静态文案无限重复并控制在系统待处理通知数量上限内。应用回到前台时刷新滚动窗口。
- 报表翻页同步选择锚点；首页进度从同一组当天非取消任务计算；历史、无障碍和 UI 测试做局部修正。
- 备份新增字段保持可选并沿用 schema version 1，恢复时对旧备份使用安全默认值，同时补齐规则和任务输入校验。

## 验收

- 固定任务改期后再次生成，不出现原日期重复任务。
- Deadline 或分值单独调整后编辑规则，调整实例仍保留。
- 通知能覆盖未来日期，不重复使用旧日文案，且只清理应用托管请求。
- 报表切换周期保留当前浏览日期；今日进度包含当天临时任务。
- 构建、全部单元测试和全部 UI 测试通过。
